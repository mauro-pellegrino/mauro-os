## What This Subtype Is

A framework for a content or ad format. The magnet teaches the operator how to think about and execute a specific format (podcast ads, hooks, scripts, scaling playbooks). Higher abstraction than a prompt pack, lower specificity than a case study.

Subtype identity: systems thinking, memorable mental model, repeatable production rules.

**Top performers shipped:**
- FORMATS keyword (X): 41K views — "$100k/day Meta Ads Formats in 2026" (5 stealth creative formats)
- HOOK keyword (LinkedIn): 15.2K impressions — "The Ultimate Hook System for Scaling Ad Creatives in 2025"
- Script keyword (LinkedIn): 11.9K impressions — "Scriptwriting $100k Ads (NOT UGC)"
- GEM keyword (LinkedIn): 11.8K impressions — "Why Brands Have Been Struggling" (GEM playbook)

If the framework doesn't have a memorable name yet, propose 2-3 names before drafting. Names that have shipped well: "Stealth Creatives", "Ultimate Hook System", "GEM playbook", "5 Formats".

## Asset Structure

The magnet ships as a Notion page, Gamma deck, Canva slides, or LinkedIn doc.

### Title format

`[Catchy framework name or outcome] for [Audience / Context]`

Variants that have shipped:
- "$100k/day Meta Ads Formats in 2026"
- "The Ultimate Hook System for Scaling Ad Creatives in 2025"
- "Scripting $100k Ads (NOT UGC)"
- "Scaling on Meta in 2026 requires a NEW playbook" (GEM)

Titles always anchor on either a dollar number, a year, or a contrarian stance ("NOT UGC"). Generic titles ("How to write hooks") do not ship.

### Subtitle / opening framing

`The ideal [thing] to scale your brand in [year] with [specific differentiator]`

or

`Why a [Quality X] Process Should Be Your Priority`

The opening framing earns the reader's attention before diving into the framework.

### Body structure

```
[Framework name + 1-sentence pitch]

PART 1: [Component name]
  [Definition / what it is in 2-3 lines]
  [Why it works / mechanism]
  [Worked example or visual]

PART 2: [Component name]
  ...

[Repeat for 4-6 parts total]

CLOSING SECTION: [Where this fits in the bigger picture / what to do next]
```

For the FORMATS magnet specifically, the body is a list of the 5 formats with one page per format:
- Debate Style Ads
- Podcast-Style Ads
- Spokesperson Ads
- Skit/Humor Ads
- Expert/Authority VSL

Each format gets: definition, scripting structure, awareness-level mapping, real example.

### Rules for each component

- Named in 1-3 words (memorable, not descriptive)
- Defined in 2-3 lines max (definitions live in the asset, not the post)
- Mechanism explained (why this component works, grounded in real client data)
- Worked example or visual reference for every component
- Where it fits in the broader system (don't leave components floating)

### Before/After illustration (optional but powerful)

When the framework flips an old way of thinking, include a before/after section. The GEM post shipped this as the cover image:

```
BEFORE
- [Old behavior 1]
- [Old behavior 2]
- [Old behavior 3]

AFTER
- [New behavior 1]
- [New behavior 2]
- [New behavior 3]
```

Use this for any framework that's positioned as "the new playbook" or "what most brands get wrong".

## Post Hook Formulas (Subtype-Specific)

`_master.md` has the universal hook formulas. For framework subtypes specifically, the patterns that have worked:

### X hooks
- **"Your [thing] [does problem], and that's [killing X]"** (FORMATS: "Your ads look like ads, and that's killing your performance.")
- Less commonly the tool-led "[Tool] is INSANE" formula (frameworks aren't always tool-anchored)

### LinkedIn hooks
- **"[Component] is the 80/20 of [thing]"** (HOOK: "Hooks are the 80/20 of ANY ad.")
- **"You need to [contrarian action]"** (Script: "You need to create ads that don't look like ads.")
- **"[Thing] requires a NEW playbook"** (GEM: "Scaling on Meta in 2026 requires a NEW playbook.")

### Setup line

The framework post body usually includes a contrast or stakes line in the setup paragraph:
- "After $107M+ in managed Meta ad spend, we broke down the 5 creative formats..."
- "After generating $100M+ on Meta ads, I built a system for..."
- "We've changed how we conduct our 7/8-fig clients' creative strategy."

The setup positions the framework as earned, not theoretical.

### Bullet structure (the "Inside the guide:" list)

- Our full breakdown on each [component] and why it works
- The exact [structure / scripting / process] we use for each one
- How each [component] maps to different [awareness levels / scenarios] so you know when to use what
- Real examples from accounts we've scaled with these

The bullets always include: a breakdown + a process + a mapping rule + real examples. That's the 4-part promise.

## Anti-Patterns (Specific to This Subtype)

- Generic framework names ("The Ad Scaling System") — needs to be specific or contrarian to be memorable
- Components without mechanisms — "what" without "why" reads as a listicle, not a framework
- Component count outside 4-6 (5 is the proven baseline) — past 6 operators stop tracking, fewer than 4 feels thin
- Skipping worked examples — frameworks without examples sound theoretical
- Hook formulas without setup — "Hooks are the 80/20 of ANY ad" works because it's followed by the $100M+ anchor in paragraph 2. Skipping the anchor weakens the hook.
- Treating this like a prompt pack — frameworks are systems thinking, not paste-ready commands
