## What This Subtype Is

A package of AI prompts (Claude, GPT, Midjourney, Veo, etc.) or a curated swipe file (hook bank, ad library, script structures) that the operator can paste, run, or copy on Monday morning.

Subtype identity: tactical, file-format, ready-to-execute. Lower abstraction than a framework, higher specificity than a case study.

- AI keyword: 73.2K views (GPT Image 2 Static Ads Prompt File)
- MAP keyword: 40.6K views (Claude Creative Strategy)
- Claude keyword: 45.5K views (Claude Research for Winning Ads)

## Input Form

Before drafting, get these from Mauro:

1. **Tool** the prompts are for (Claude, GPT, Midjourney, Veo, Sora, n8n, etc.)
2. **Use case / domain** (scriptwriting, creative strategy, research, hook generation, persona mapping, etc.)
3. **Number of prompts + page format** — three patterns that ship:
   - **5 pages × 2 prompts each** (= 10 prompts total). Most common when prompts are medium-length and guided across phases.
   - **10 standalone prompts.** Used when prompts are short and paste-ready as a flat list.
   - **5 pages × 1 long prompt each** (= 5 prompts total). Used when each prompt is dense (30+ lines) and warrants its own page.

   The rule: match the format to how the team actually uses the prompts. Long prompts get their own page. Short related prompts pair up. Don't default — ask which pattern fits.
4. **Grouping structure** (let Claude propose pillars / phases, or Mauro specifies)
5. **Worked examples required** (yes by default. Always include in the asset, mention in the post bullets.)
6. **Attached file(s)** (e.g., the SCRIPT magnet shipped with `anti-ai-script.md` attached. Note any companion files.)
7. **Target client tier** (informs the subtitle: "scale our clients past $1M/month on Meta", "for our 7/8-figure clients")
8. **Trigger keyword** (3-7 chars, ALL CAPS, topic-related. Examples shipped: SCRIPT, MAP, AI, Claude.)
9. **Trend signal** (is a specific tool having a moment right now? If yes, lean the hook into that. Use trendjack alert feed when live.)

If any of these are missing, ask before drafting. Don't guess on tool name or keyword.

## Asset Structure

The magnet itself ships as a Notion page (most common) or an attached `.md` file.

### Title format

The title renders large in the X / LinkedIn feed. Treat it as a second hook (after the post hook itself). Spend real attention on this. Generic titles cost feed performance.

Two title patterns have shipped well:

**Pattern A: Tool-led, prompt-only magnets**
`[Tool] [Use Case] for Winning Ads`
- "Claude Scriptwriting Stack for Winning Ads"
- "Claude Creative Strategy for Winning Ads"
- "Claude Research for Winning Ads"

Use when the magnet centers on a workflow or use case applied across multiple formats.

**Pattern B: Tool-led, format-specific magnets**
`[Tool] [Format] Prompt File`
- "GPT Image 2 Static Ads Prompt File"

Use when the magnet centers on a specific output format (static ads, video ads, scripts, hooks).

**Always propose 3 title options for Mauro to pick.** Never ship a single title without alternatives. The cost of a generic title is too high to commit on one shot.

### Subtitle format

`[N] prompts we use to [outcome that maps to scale] past $[X]M/month on Meta.`

Variants:
- "10 prompts we use to write the scripts that scale our clients past $1M/month on Meta."
- "10 prompts we use to map the creative strategy that scaled multiple clients past $1M/month on Meta. Same 5-pillar process across $107M+ in managed ad spend."

The subtitle always includes a number anchor and a scale anchor. No subtitle without these.

### Body structure

```
SETUP ONCE
[1-2 short setup steps before the prompts, e.g. paste this brand brief, create a project, install MCP]

PROMPT GROUP 1: [Group name] (Prompts 1 & 2 if 10-prompt format)
  Prompt 1: [Purpose] → [paste-ready prompt]
  Prompt 2: [Purpose] → [paste-ready prompt]
  Worked example: [Real output from running these prompts]

PROMPT GROUP 2: [Group name] (Prompts 3 & 4)
  ...

[Repeat for 4-5 groups total in a 10-prompt magnet]

OR for a 5-prompt magnet (Claude Research format):
  Prompt 1: CUSTOMER RESEARCH PROMPT → [paste-ready]
  Prompt 2: COMPETITOR RESEARCH PROMPT → [paste-ready]
  Prompt 3: REVIEW MINING PROMPT → [paste-ready]
  Prompt 4: WINNING AD & COMPETITOR SCRIPT MINING PROMPT → [paste-ready]
  Prompt 5: CONTENT & COMMENT MINING (TIKTOK + YOUTUBE) PROMPT → [paste-ready]
```

### Rules for each prompt

- Paste-ready. The operator should not have to fill in [BRAND] or [PRODUCT] tokens unless explicitly instructed in a setup step.
- Specific. Generic prompts ("write 5 hooks") underperform specific prompts ("write 5 hooks in podcast-ad format for a 7-figure skincare brand targeting women 35-55 who already use retinol").
- Grounded in real client work. The prompt should reflect how Growthub actually uses Claude, not a generic prompt-engineering exercise.
- Single-output focused. One prompt = one deliverable. No mega-prompts that try to do 5 things.

### Worked examples block (mandatory)

After the prompt block, ship a worked example: real input + real output from running the prompt. Anonymized if it includes a client.

The worked example is what separates this from a generic prompt library. Operators want to see the output before they trust the prompt.

## Post Hook Formulas (Subtype-Specific)

`_master.md` has the universal X and LinkedIn hook formulas. For prompt-swipe-file specifically, the highest-performing hook formula is:

**"[Tool] is INSANE for [use case]"** or **"[Tool] is CRAZY good"** when the tool is novel and trending.

Used variants shipped:
- "Claude is INSANE for ad scriptwriting." (SCRIPT)
- "Claude is INSANE for ad creative strategy." (MAP)
- "Claude is INSANE for ad creative research." (Claude)
- "GPT Images 2.0 is CRAZY good." (AI)

The post body for this subtype almost always includes a "I've built a file with [N] prompts" or "I've built a file with [N] prompts we're testing" line in the setup. That phrase is the bridge between the hook and the bullet list.

### Bullet structure (the "Inside the mini-guide:" list)

For 10-prompt magnets:
- The [N] prompts
- [Companion file name or concept, if attached]
- Worked examples

For 5-prompt magnets:
- The [N] prompts
- [Tool] setup walkthrough
- Worked examples

Bullets are short. The post is not the magnet — it's a teaser for what's inside.

## Anti-Patterns (Specific to This Subtype)

- Generic prompts ("write me 5 hooks") — paste-able by anyone with ChatGPT, doesn't earn the magnet
- Prompt walls of text without grouping — the magnet is the grouping, not the raw prompts
- Skipping worked examples — operators lose trust in the prompts immediately
- Subtitle without a scale anchor — looks like a generic prompt pack from anyone
- Hook reuse without a real trend behind it — replicating "GPT Images 2.0 is CRAZY good" 3 months after the tool peaked will underperform, even with the same formula
