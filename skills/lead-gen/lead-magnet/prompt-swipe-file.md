## What This Subtype Is

A package of AI prompts (Claude, GPT, Midjourney, Veo, etc.) or a curated swipe file (hook bank, post structures, script structures) that the agency owner can paste, run, or copy on Monday morning.

Subtype identity: tactical, file-format, ready-to-execute. Lower abstraction than a framework, higher specificity than a case study.

**Track record note:** tool-led prompt files were the top autodm performers on the agency account Mauro ran before this brand, and the "Claude is INSANE for [X]" opener is a proven pattern for Claude-based auto-DM posts. This is the default first subtype to ship on this account. Log each magnet's performance in `brand/` as its own data accrues.

## Input Form

Before drafting, get these from Mauro:

1. **Tool** the prompts are for (Claude, GPT, Midjourney, Veo, Sora, n8n, etc.)
2. **Use case / domain** (turning call transcripts into content, lead magnet generation, article writing, audience research, client reporting, etc.)
3. **Number of prompts + page format** — three patterns that work:
   - **5 pages × 2 prompts each** (= 10 prompts total). Most common when prompts are medium-length and guided across phases.
   - **10 standalone prompts.** Used when prompts are short and paste-ready as a flat list.
   - **5 pages × 1 long prompt each** (= 5 prompts total). Used when each prompt is dense (30+ lines) and warrants its own page.

   The rule: match the format to how the prompts are actually used. Long prompts get their own page. Short related prompts pair up. Don't default — ask which pattern fits.
4. **Grouping structure** (let Claude propose pillars / phases, or Mauro specifies)
5. **Worked examples required** (yes by default. Always include in the asset, mention in the post bullets.)
6. **Attached file(s)** (note any companion files that ship with the magnet)
7. **Target reader tier** (informs the subtitle; default is the established agency owner per `brand/audience.md`)
8. **Trigger keyword** (3-7 chars, ALL CAPS, topic-related. Examples: SCRIPT, ENGINE, AI, CLAUDE.)
9. **Trend signal** (is a specific tool having a moment right now? If yes, lean the hook into that.)

If any of these are missing, ask before drafting. Don't guess on tool name or keyword.

## Asset Structure

The magnet itself ships as a Notion page (most common) or an attached `.md` file.

### Title format

The title renders large in the X / LinkedIn feed. Treat it as a second hook (after the post hook itself). Spend real attention on this. Generic titles cost feed performance.

Two title patterns that work:

**Pattern A: Tool-led, workflow magnets**
`[Tool] [Use Case] for [Outcome]`
- "Claude Content Engine Stack for Agency Inbound"
- "Claude Call-to-Content System for Agency Owners"

Use when the magnet centers on a workflow or use case applied across multiple formats.

**Pattern B: Tool-led, format-specific magnets**
`[Tool] [Format] Prompt File`
- "Claude Lead Magnet Prompt File"
- "GPT Image Infographic Prompt File"

Use when the magnet centers on a specific output format (posts, articles, lead magnets, infographics).

**Always propose 3 title options for Mauro to pick.** Never ship a single title without alternatives. The cost of a generic title is too high to commit on one shot.

### Subtitle format

`[N] prompts I use to [outcome tied to the real engine].`

Variants:
- "10 prompts I use to turn one week of client calls into a month of content."
- "10 prompts behind the content engine I run daily for a real B2B agency."

The subtitle always includes a number anchor (N prompts) and a proof anchor (the live engine, or a signed-off number per `_master.md`). No subtitle without these.

### Body structure

```
SETUP ONCE
[1-2 short setup steps before the prompts, e.g. paste this brand brief, create a project, install MCP]

PROMPT GROUP 1: [Group name] (Prompts 1 & 2 if 10-prompt format)
  Prompt 1: [Purpose] → [paste-ready prompt]
  Prompt 2: [Purpose] → [paste-ready prompt]
  Worked example: [Real output from running these prompts]

PROMPT GROUP 2: [Group name] (Prompts 3 & 4)
  ...

[Repeat for 4-5 groups total in a 10-prompt magnet]

OR for a 5-prompt magnet (flat research/workflow format):
  Prompt 1: [PURPOSE] → [paste-ready]
  Prompt 2: [PURPOSE] → [paste-ready]
  ...
```

### Rules for each prompt

- Paste-ready. The reader should not have to fill in [AGENCY] or [CLIENT] tokens unless explicitly instructed in a setup step.
- Specific. Generic prompts ("write 5 posts") underperform specific prompts ("write 5 posts in build-log format for an SEO agency owner documenting a client migration, speaking to other agency owners").
- Grounded in real work. The prompt should reflect how the live engine actually uses the tool, not a generic prompt-engineering exercise.
- Single-output focused. One prompt = one deliverable. No mega-prompts that try to do 5 things.

### Worked examples block (mandatory)

After the prompt block, ship a worked example: real input + real output from running the prompt. Anonymized if it includes a client.

The worked example is what separates this from a generic prompt library. Operators want to see the output before they trust the prompt.

## Post Hook Formulas (Subtype-Specific)

`_master.md` has the universal X and LinkedIn hook formulas. For prompt-swipe-file specifically, the highest-performing hook formula is:

**"[Tool] is INSANE for [use case]"** or **"[Tool] is CRAZY good"** when the tool is novel and trending.

Example variants for this brand's lane:
- "Claude is INSANE for turning client calls into content."
- "Claude is INSANE for building lead magnets."
- "Claude is INSANE for agency audience research."

The post body for this subtype almost always includes a "I've built a file with [N] prompts" or "I've built a file with [N] prompts we're testing" line in the setup. That phrase is the bridge between the hook and the bullet list.

### Bullet structure (the "Inside the mini-guide:" list)

For 10-prompt magnets:
- The [N] prompts
- [Companion file name or concept, if attached]
- Worked examples

For 5-prompt magnets:
- The [N] prompts
- [Tool] setup walkthrough
- Worked examples

Bullets are short. The post is not the magnet — it's a teaser for what's inside.

## Anti-Patterns (Specific to This Subtype)

- Generic prompts ("write me 5 posts") — paste-able by anyone with ChatGPT, doesn't earn the magnet
- Prompt walls of text without grouping — the magnet is the grouping, not the raw prompts
- Skipping worked examples — operators lose trust in the prompts immediately
- Subtitle without a proof anchor — looks like a generic prompt pack from anyone
- Hook reuse without a real trend behind it — a tool-hype hook months after the tool peaked will underperform, even with the same formula
